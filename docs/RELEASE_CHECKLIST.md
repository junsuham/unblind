# 배포 체크리스트

## 배포 전

- [ ] 변경된 Supabase 마이그레이션을 먼저 검토하고 백업을 확인했다.
- [ ] `npm run check`가 통과한다.
- [ ] `npm audit --omit=dev` 결과를 검토했다.
- [ ] Vercel의 필수 환경 변수가 Production, Preview에 올바르게 구분되어 있다.
- [ ] Google OAuth 허용 URL과 Supabase Site URL이 `https://unbd.vercel.app` 기준으로 일치한다.
- [ ] 개인정보처리방침의 운영자 이름, 연락처, 위탁·국외 처리 상세를 실제 계약 기준으로 검토했다.

## 배포 직후

- [ ] `npm run smoke:production`이 통과한다.
- [ ] iPhone 홈 화면 PWA를 완전히 종료한 뒤 다시 열어 시작 화면, 로그인 유지, 하단 안전영역을 확인한다.
- [ ] 홈 → 게시판 → 게시글 → 댓글 → 알림 → 내 정보 이동이 즉시 반응한다.
- [ ] 새 글 초안 복원, 이미지 3장 업로드·삭제, 위치·찬양 첨부를 확인한다.
- [ ] 댓글, 공감, 저장, 신고와 관리자 조치가 각각 한 번만 반영된다.
- [ ] 최신 서비스 워커에서만 업데이트 안내가 표시되고 업데이트 버튼이 작동한다.
- [ ] 오프라인 안내와 온라인 복구 안내가 정상 표시된다.
- [ ] iPhone PWA 푸시를 허용하고 테스트 알림을 수신한다.

## 되돌리기 기준

- 로그인 불가, 데이터 손실, 게시물 중복 생성, 관리자 권한 오류, 흰 화면이 발생하면 즉시 이전 정상 배포로 되돌린다.
- 데이터베이스 마이그레이션은 되돌리기 SQL을 별도로 검토한 경우에만 역적용한다.
- 장애 배포를 되돌린 뒤 동일 커밋을 재배포하지 말고 원인과 검증 결과를 기록한다.
