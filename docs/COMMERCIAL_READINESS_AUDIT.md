## 1차 감사 결과

초기 코드 기준 점수는 74/100이었다. 핵심 커뮤니티·관리자·신앙 기능은 갖췄지만 다음 문제가 상용화 품질을 낮췄다.

- 라이트 모드 전체 배경이 강한 오렌지여서 장시간 읽기에 부담이 컸다.
- 화면에 따라 상단 제목이 중복되고, 홈과 게시판의 정보 계층이 달랐다.
- 게시판 정렬 UI가 실제 정렬을 수행하지 않았고 긴 목록에 페이지 구분이 없었다.
- 9~10px 글씨와 44px 미만 조작부가 일부 남아 있었다.
- iOS에서 16px 미만 입력란에 초점을 두면 화면이 자동 확대될 수 있었다.
- 서비스 워커가 개발 중 과거 화면을 남겨 최신 수정이 반영되지 않은 것처럼 보일 수 있었다.
- 웹/PWA 가입 화면에서 생년월일을 직접 입력할 수 있어 소셜 연령 확인을 우회할 수 있었다.
- 내 정보에서 기도 이력과 주요 계정 메뉴로 가는 동선이 부족했다.
- 로그인 화면 성경 구절 표기가 본문과 맞지 않았고 일부 용어·말줄임표가 일관되지 않았다.

## 반영한 고도화

### 화면과 접근성

- 라이트 모드 본문을 따뜻한 중성색으로 바꾸고 오렌지는 상단 브랜드와 주요 행동에 집중했다.
- 320×568, 375×812, 390×844, 430×932와 가로 화면에서 가로 넘침을 검사했다.
- iOS 입력 자동 확대 방지 규칙을 추가하되 사용자의 손가락 확대 기능은 제한하지 않았다.
- 조작부를 원칙적으로 44px 이상으로 올리고, 9~10px 본문 글씨를 11px 이상으로 정리했다.
- 키보드 포커스 링, 현재 탭 `aria-current`, 실제 페이지 제목과 뒤로가기를 보강했다.
- 라이트·다크 모드의 카드·탭·오류색을 의미 기반 디자인 토큰으로 통합했다.

### 핵심 흐름과 효율

- 홈·게시판·게시글의 상단 구조를 하나로 합치고 중복 제목을 제거했다.
- 게시판 최신순·인기순을 실제 링크로 동작시키고 20개 단위 페이지 이동을 추가했다.
- 상시 노출되던 글쓰기 말풍선을 제거해 콘텐츠 가림과 반복 안내를 줄였다.
- 내 정보에 계정·알림·도움말, 기도한 글, 연속 기도 기록을 추가했다.
- Google 로그인 화면에 이용약관·개인정보처리방침 진입점을 추가했다.

### 신뢰와 운영

- Google People API의 생년월일 권한을 요청하고 로그인 콜백에서 연령을 검증한다.
- 프로필 UI와 저장 API 모두 직접 입력한 생년월일을 받지 않고 서버에 검증된 Google 값만 허용한다.
- 개발 환경의 오래된 서비스 워커와 정적 캐시를 자동 제거한다.
- 안전한 진입 화면에서는 새 서비스 워커를 자동 적용하고 작성 중인 화면에서는 명시적으로 안내한다.
- 관리자 초기 지정 이메일을 Production·Preview 환경에 등록했다.
- 내부 오류 문자열 대신 사용자가 이해할 수 있는 복구 안내를 표시한다.
- Next.js를 16.2.11로 패치하고 이미지 처리 의존성을 안전한 버전으로 고정해 운영 의존성 고위험 취약점을 0건으로 정리했다.

## 코드 밖에서 완료해야 할 공개 운영 조건

95점은 앱 코드와 폐쇄형 베타 출시 기준이다. 불특정 다수를 대상으로 한 공개 상용 서비스는 아래 실운영 항목을 완료하기 전까지 82/100으로 본다.

1. 개인정보처리방침에 실제 운영 주체, 개인정보 책임자, 공개 문의 주소를 확정한다.
2. Google OAuth 동의 화면에 People API 생년월일 범위를 등록하고 실제 신규 계정으로 재동의 흐름을 확인한다.
3. 신고 1차 응답 시간, 긴급 위험 대응 담당자, 이의 제기 절차를 실제 인력 기준으로 확정한다.
4. 실제 사용자 데이터로 월간 복구 훈련, 부하 검사, 푸시 도달률과 오류율 기준을 운영한다.
5. 콘텐츠·커뮤니티 규모는 기능으로 대체할 수 없으므로 초기 운영진의 게시물·기도 응답·찬양 큐레이션 계획을 실행한다.

이 항목은 운영자 신원·연락처와 실제 인력 결정이 필요하므로 코드가 임의로 채울 수 없다.
